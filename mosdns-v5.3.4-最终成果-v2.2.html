3.1 时区修复 (v2.2 — 2026-05-19)

问题诊断

Alpine 基础镜像默认 UTC 时区。虽然 docker-compose.yml 设置了 environment: TZ=Asia/Shanghai，但容器内缺少 /usr/share/zoneinfo/ 时区数据（需 tzdata 包）。musl libc 找不到时区文件 → 回退 UTC。cron 0 3 * * * 实际在 UTC 3:00 = CST 11:00 执行，规则更新时间戳偏移 8 小时。

修复方案

文件改动
Dockerfileapk add dcron curl → apk add dcron curl tzdata（+~2MB）
docker-compose.yml新增 environment: TZ=Asia/Shanghai

原理：Alpine musl libc 进程启动时自动读取 $TZ 环境变量 → 查 /usr/share/zoneinfo/Asia/Shanghai → CST +0800。纯 OS 级别行为，无需脚本干预。

验证结果

检查项修复前修复后
容器时区UTC +0000CST +0800
容器时间00:23 UTC08:25 CST
cron 执行时刻UTC 03:00 = CST 11:00CST 03:00
文件时间戳11:0003:00
TZ 动态切换固定 UTCUTC/JST/EDT 均可
镜像体积111MB（基本无变化）
