2. 版本变更 (v1.0 → v2.0)

v2.0 改动摘要

改动v1.0v2.0原因
dial_addr 全部上游硬编码 dial_addr 全部删除 dial_addr 强制连接错误 IP → DoH 失效；DNS 自然解析更可靠
国外主备 Google 主 → Cloudflare 备 Cloudflare 主 → Google 备 延迟：Cloudflare 7ms vs Google 17ms (2.4x)
协议精简 每上游 4 条 addr (2 DoH + 2 DoT)，含双 IP 冗余 每上游 1 DoH + 1 DoT 无 dial_addr 时 DNS 自然解析提供多 IP；h3:// 比 DoT 慢 5x 不采用
dat_exec.yaml ttl_1m 中间标签 移除 ttl_1m，精简标签 未被引用，冗余
main_sequence 顺序 cn → !cn → gfw gfw → !cn → cn 被墙域名优先走国外上游
自更新 无自动更新，规则手动维护 dcron 容器内定时更新 每24h检查 Loyalsoldier，自动下载+解包+重启

4.3 dns.yaml v2.0 — Cloudflare 主 · 去 dial_addr

# ============================================================
# dns.yaml — 上游 DNS / ECS / Reject
# DoH + DoT 并发，仅 IPv4
# 2026-05-13: 去 dial_addr，Cloudflare主/Google备
# ============================================================

plugins:
  # ── Cloudflare DoH + DoT（主，avg 7ms）──
  - tag: cloudflare
    type: forward
    args:
      concurrent: 3
      upstreams:
        - addr: "https://cloudflare-dns.com/dns-query"
        - addr: "tls://1dot1dot1dot1.cloudflare-dns.com"
          enable_pipeline: true

  # ── Google DoH + DoT（备，avg 17ms）──
  - tag: google
    type: forward
    args:
      concurrent: 3
      upstreams:
        - addr: "https://dns.google/dns-query"
        - addr: "tls://dns.google"
          enable_pipeline: true

  # ── 国外回退：Cloudflare → Google ──
  - tag: dns_nocn
    type: fallback
    args:
      primary: cloudflare
      secondary: google
      threshold: 500
      always_standby: true

  # ── AliDNS DoH + DoT（v4，国内主）──
  - tag: ali
    type: forward
    args:
      concurrent: 3
      upstreams:
        - addr: "https://dns.alidns.com/dns-query"
        - addr: "tls://dns.alidns.com"
          enable_pipeline: true
          insecure_skip_verify: true

  # ── DNSPod DoH + DoT（v4，国内备）──
  - tag: dnspod
    type: forward
    args:
      concurrent: 3
      upstreams:
        - addr: "https://doh.pub/dns-query"
        - addr: "tls://dot.pub"
          enable_pipeline: true

  # ── 国内回退：AliDNS → DNSPod ──
  - tag: dns_cn
    type: fallback
    args:
      primary: ali
      secondary: dnspod
      threshold: 500
      always_standby: true

  # ── ECS 处理 ──
  - tag: no_ecs
    type: ecs_handler
    args:
      forward: false
      preset: ""
      send: false
      mask4: 24
      mask6: 48

  - tag: ecs_cn
    type: ecs_handler
    args:
      forward: false
      preset: "202.120.2.100"
      send: false
      mask4: 24
      mask6: 48

  # ── Reject 响应 ──
  - tag: reject_3
    type: sequence
    args:
      - exec: reject 3

  - tag: reject_2
    type: sequence
    args:
      - exec: reject 2

  - tag: reject_5
    type: sequence
    args:
      - exec: reject 5

4.4 dat_exec.yaml 分流核心

plugins:
  # ── 数据源 (domain_set / ip_set) ──
  - tag: geoip_private
    type: ip_set
    args:
      files: ["/etc/mosdns/dat/geoip_private.txt"]

  - tag: geoip_cn
    type: ip_set
    args:
      files: ["/etc/mosdns/dat/geoip_cn.txt"]

  - tag: geosite_cn
    type: domain_set
    args:
      files: ["/etc/mosdns/dat/geosite_cn.txt"]

  - tag: geosite_ads_all
    type: domain_set
    args:
      files: ["/etc/mosdns/dat/geosite_category-ads-all.txt"]

  - tag: geosite_gfw
    type: domain_set
    args:
      files: ["/etc/mosdns/dat/geosite_gfw.txt"]

  - tag: geosite_location_not_cn
    type: domain_set
    args:
      files: ["/etc/mosdns/dat/geosite_geolocation-!cn.txt"]

  # ── 双缓存 ──
  - tag: cache_lan
    type: cache
    args:
      size: 8192
      lazy_cache_ttl: 86400

  - tag: cache_wan
    type: cache
    args:
      size: 131072
      lazy_cache_ttl: 86400
      dump_file: ./cache.dump
      dump_interval: 120

  # ── TTL 分级 ──
  - tag: ttl_1h
    type: sequence
    args: [{ exec: ttl 3600 }]

  - tag: ttl_5m
    type: sequence
    args: [{ exec: ttl 300 }]

  # ── 结果处理 ──
  - tag: pre_handle
    type: sequence
    args: [{ exec: $ttl_1h }, { exec: accept }]

  - tag: main_handle
    type: sequence
    args: [{ exec: $ttl_5m }, { exec: accept }]

  - tag: has_resp_pre
    type: sequence
    args: [{ matches: has_resp, exec: goto pre_handle }]

  - tag: has_resp_main
    type: sequence
    args: [{ matches: has_resp, exec: goto main_handle }]

  # ── 国内查询（ECS + 反污染）──
  - tag: query_cn
    type: sequence
    args:
      - exec: $ecs_cn
      - exec: $dns_cn
      - matches: "!resp_ip $geoip_cn"
        exec: drop_resp

  # ── 国外查询（反污染）──
  - tag: query_nocn
    type: sequence
    args:
      - exec: $dns_nocn
      - matches: "resp_ip $geoip_cn"
        exec: drop_resp

  # ── 内网查询 ──
  - tag: query_lan
    type: sequence
    args:
      - exec: $cache_lan
      - matches: has_resp
        exec: return
      - exec: $no_ecs
      - exec: $dns_cn

  # ── 其他情况 ──
  - tag: query_other
    type: sequence
    args:
      - exec: $no_ecs
      - exec: $dns_nocn

  # ── 拒绝处理 ──
  - tag: reject_qtype65
    type: sequence
    args: [{ exec: query_summary reject_qtype65 }, { exec: $reject_3 }]

  - tag: reject_null_domain
    type: sequence
    args: [{ exec: query_summary reject_null_domain }, { exec: $reject_3 }]

  - tag: reject_ad
    type: sequence
    args: [{ exec: query_summary reject_ad }, { exec: $reject_3 }]

  # ── 预处理 ──
  - tag: pre_sequence
    type: sequence
    args:
      - matches: "qtype 65"
        exec: $reject_qtype65
      - matches: "qname keyword::"
        exec: $reject_null_domain
      - matches: "qtype 12"
        exec: $query_other
      - matches: "qtype 255"
        exec: $query_other
      - matches: "ptr_ip $geoip_private"
        exec: $query_lan
      - exec: jump has_resp_pre

  # ── 主流程 ──
  - tag: main_sequence
    type: sequence
    args:
      - matches: "qname $geosite_ads_all"
        exec: $reject_ad
      - exec: $cache_wan
      - exec: jump has_resp_main
      - matches: "qname $geosite_gfw"
        exec: $query_nocn
      - exec: jump has_resp_main
      - matches: "qname $geosite_location_not_cn"
        exec: $query_nocn
      - exec: jump has_resp_main
      - matches: "qname $geosite_cn"
        exec: $query_cn
      - exec: jump has_resp_main
      - exec: $no_ecs
      - exec: $query_nocn
      - exec: jump has_resp_main

  # ── 顶层 ──
  - tag: sequence
    type: sequence
    args:
      - exec: $pre_sequence
      - exec: $main_sequence

5. 协议对比分析

5.1 延迟对比（Cloudflare 上游，3 域名平均值）

5.3 dial_addr 问题分析

dial_addr 值实际连接问题
1.1.1.1 (Cloudflare DoH) 连接 1.1.1.1:443 1.1.1.1:443 不提供 DoH 服务 → 返回错误数据
8.8.8.8 (Google DoH) 连接 8.8.8.8:443 强制 SNI=dns.google 但 IP 是 8.8.8.8 → SNI 不匹配
223.5.5.5 (AliDNS DoH) 连接 223.5.5.5:443 可用但失去多 IP 负载均衡
无 dial_addr DNS 自然解析 cloudflare-dns.com → 4 IP SNI 正确 + 自动负载均衡 ✅

结论：删除所有 dial_addr。cloudflare-dns.com 自然解析到 104.16.248.249/104.16.249.249/1.1.1.1/1.0.0.1，多 IP 自动负载 + SNI 正确。

8. 反污染验证协议

IP 归属验证标准

步骤命令/方法说明
1. whoiswhois <IP>确认 AS 归属和注册组织
2. SSL 证书openssl s_client -connect <IP>:443 -servername <域名>验证 CN/SAN 是否匹配目标域名
3. HTTP 访问curl -k -H "Host: <域名>" https://<IP>/检查是否返回合法内容
4. AS 验证whois -h whois.radb.net AS<编号>确认 AS 归属组织

严禁凭 IP 段推测是否被污染。遇到可疑 IP 必须通过以上步骤验证。已验证案例：57.145.12.1 → AS32934 Facebook 香港 CDN，SSL CN=*.facebook.com，非污染。
